ImageDownloadRetryPolicy

class ImageDownloadRetryPolicy @JvmOverloads constructor(val maxAttempts: Int = DEFAULT_MAX_ATTEMPTS, val backoffStateProviderFactory: () -> IBackoffStateProvider = { defaultBackoffStateProvider() }, val sleeper: (Long) -> Unit = ::defaultSleep, val isOnMainThread: () -> Boolean = { Looper.myLooper() == Looper.getMainLooper() })

Executes a download attempt with exponential backoff between failures.

Retries only when attempt returns null. Successful (non-null) results are returned immediately without sleeping. Designed for short-lived push image fetches where total wall time must stay bounded.

Backoff delays come from backoffStateProviderFactory, which by default builds an ExponentialBackoffStateProvider using ServerConfigStorageProvider request-backoff values (DefaultRequestBackoffConfig from v3/data).

Parameters

maxAttempts

Maximum number of download attempts, including the first try. Must be at least 1.

backoffStateProviderFactory

Creates the backoff provider used for a single execute call. A new provider is created per execution so updated server config values are picked up.

sleeper

Blocks for the given delay in milliseconds between failed attempts. Defaults to Thread.sleep.

isOnMainThread

Returns true when the current thread is the main thread. Sleep is skipped and remaining retries are aborted when this returns true.
