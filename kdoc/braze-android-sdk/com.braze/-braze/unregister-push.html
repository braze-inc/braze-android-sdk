unregisterPush

open override fun unregisterPush(callback: (Result<Unit>) -> Unit)

Unregisters this device's push token with Braze.

Any pending requests are flushed first, then a single request is sent to the push/unregister endpoint with no retry logic. The integrator owns what to do on failure (for example, a custom retry mechanism). When the failure's com.braze.push.BrazePushUnregistrationException.isRetriable flag is true, implement your own retry logic with a backoff and a maximum number of attempts rather than retrying immediately or indefinitely.

On success, the locally stored push token is cleared so it is not sent again on a later request. This does not wipe other local SDK data or disable the SDK. For a full logout, use logout instead, or call unregisterPush first while the SDK is enabled, then Braze.wipeData and Braze.disableSdk on success.

Parameters

callback

Invoked once with a Result. On success the result is Result.success of Unit; on failure it is Result.failure wrapping a com.braze.push.BrazePushUnregistrationException carrying an error message and an com.braze.push.BrazePushUnregistrationException.isRetriable flag.

braze.unregisterPush { result ->
    result
        .onSuccess {
            Braze.wipeData(context)
            Braze.disableSdk(context)
        }
        .onFailure { error -> /* custom retry */}
}

private fun unregisterPush(pushToken: String?, callback: (Result<Unit>) -> Unit)

Backs both the public unregisterPush (which resolves the token from storage) and the internal logout flow (which resolves the token once and threads it in so it is not re-read from storage on every retry attempt).

When pushToken is null the token is resolved from storage; otherwise the provided, already-validated token is used.
